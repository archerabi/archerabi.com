This assembly may have been downloaded from the Web. If an assembly has been downloaded from the Web, it is flagged by Windows as being a Web file, even if it resides on the local computer. This may prevent it from being used in your project. You can change this designation by changing the file properties. Only unblock assemblies that you trust This is a common issue that many beginner windows phone developers face. In spite of searching for an hour the only solution i found for this issue was to right click on the dll and then click "Unblock" . Although this is the solution , this didn't work for me , mostly because of my stupidity. If you look at the error message carefully , it complains about the dll in the DEBUG/RELEASE folder not the project folder. After unblocking the assembly in your project , do the same for the one in the DEBUG/RELEASE folder.
